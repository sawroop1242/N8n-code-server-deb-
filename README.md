# InstaVibe (React Native, No Auth)

A frictionless React Native app matching your reference screens:
- Welcome vibe + styles feed
- Tap a style, upload your photo, generate image
- Save to cloud DB (`id`, `prompt`, `image`)
- No authentication flow

## Best free cloud DB for your case

I recommend **Supabase** for this app:
- Free tier is enough for MVP usage
- Stores text fields and image URLs cleanly
- Easy direct client integration with React Native
- SQL table exactly in your requested 3-column model (`id`, `prompt`, `image`)

> Note: For production, route writes via a backend/edge function. This template follows your direct-connect/no-auth request.

## 1) Setup

```bash
npm install
cp .env.example .env
```

Fill `.env` with your Supabase + Bytz values.

## 2) Create DB table

Run SQL in your Supabase SQL editor:

```sql
-- file: supabase/schema.sql
create table if not exists public.vibes (
  id bigint generated by default as identity primary key,
  prompt text not null,
  image text not null,
  created_at timestamptz default now()
);

alter table public.vibes enable row level security;

create policy "Public read vibes"
on public.vibes for select
using (true);

create policy "Public insert vibes"
on public.vibes for insert
with check (true);
```

## 3) Run app

```bash
npm run start
```

Use Expo Go to open on mobile.

## App flow

1. **Home tab** loads trending/new/all styles.
2. Select any style -> opens generation modal.
3. Upload your image.
4. App sends `prompt + image` to Bytz model.
5. Generated image URL is saved to Supabase `vibes`.
6. **Library tab** shows all created vibes.

## Key files

- `App.js` - Bottom tab app shell (Home/Library)
- `src/screens/HomeScreen.js` - style lists and modal trigger
- `src/components/StyleModal.js` - upload + generate + save
- `src/screens/LibraryScreen.js` - gallery from DB
- `src/lib/bytz.js` - Bytz API call helper
- `src/lib/supabase.js` - Supabase client
- `supabase/schema.sql` - table + policies
